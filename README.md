## What I've been building lately

<br/>

**[Market Regime Intelligence Engine](https://github.com/hems-00/quant)** &nbsp;

A ticker-agnostic ML pipeline that blends raw OHLCV market data with FinBERT sentiment signals to classify market risk regimes. Built with LightGBM and TimeSeriesSplit cross-validation so there's no look-ahead bias leaking into results. SHAP attribution gives the model actual explainability ‚Äî you know *why* it flagged a regime shift, not just that it did.

`Python` `LightGBM` `FinBERT` `SHAP` `Pandas` `Scikit-learn`

<br/>

**[Crypto Tracker ‚Äî Notification System](https://github.com/hems-00/money)** &nbsp;

Spring Boot backend that watches crypto prices and fires email alerts the moment user-defined thresholds are crossed. Sub-minute polling intervals, H2 for local caching, and exponential backoff on API rate limits so it doesn't fall apart at 3am. Layered architecture (Controller ‚Üí Service ‚Üí Repository) keeps it clean and extensible.

`Java` `Spring Boot` `Spring Data JPA` `Hibernate` `H2` `Spring Mail`

<br/>

**[Real-Time Concurrent Chat Application](https://github.com/hems-00/chat_app)** &nbsp;

Multi-threaded Java server handling 100+ concurrent chat sessions with synchronized DB writes and zero message loss across 10k+ stored entries. Built the full stack ‚Äî server, client, and storage ‚Äî and learned a lot about what thread contention actually looks like under load.

`Java` `JavaFX` `TCP Sockets` `SQLite` `JDBC`

<br/>

**climaX ‚Äî Movie Search Engine** &nbsp;

React app pulling live data from TMDB, with search trend tracking stored in Appwrite Cloud. The goal was less "movie app" and more "learning how to architect real-time state in React without it becoming a mess."

`ReactJS` `TMDB API` `Appwrite` `CSS Modules`
